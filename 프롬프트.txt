## STEP 1. 네이버 검색 API로 식당 데이터 수집

네이버 지역 검색 API를 사용해서 아래 식당 목록의 정보를 수집하는 Python 스크립트(collect_restaurants.py)를 작성해줘.

**수집 대상 식당 목록 (총 44개)**
아래 식당들을 각각 "식당명 + 중구" 키워드로 네이버 지역 검색 API에 검색해서 정보를 수집할 것.

술술돼지, 누나홀닭, 함평집, 전주다대기, 담원순대, 명동순대국, 우림정, 오늘통닭, 슬로우캘리,
교동전선생, 십원집, 신의주찹쌀순대, 가배도, 당당, 완백부대찌개, 오한수 우육면가, 고래카레,
유브유부, 서평옥, 브라운돈까스, 아이엠티, 뱃남미식, 포항집, 멘츠루, 도우보이즈클럽,
정동칼국수, 유방녕의웍, 승환네 닭한마리, 정원레스토랑 어반가든, 방일해장국, 카츠하이드,
리나스 숭례문, 이여곰탕, 백채김치찌개, 달인명동칼국수보쌈, 쪽삼상회, 멘무샤,
송원스키야키샤브샤브, 남경, 굽돌집, 김명자굴국밥, 북창동개성칼만두, 무교동북어국집,
이타마에스시 광화문디타워점

**API 호출 방식**
- 엔드포인트: https://openapi.naver.com/v1/search/local.json
- 헤더에 X-Naver-Client-Id, X-Naver-Client-Secret 사용
- 환경변수에서 키를 읽어옴: os.environ.get("NAVER_CLIENT_ID"), os.environ.get("NAVER_CLIENT_SECRET")
- 식당마다 개별 API 호출, 검색 결과 1순위 항목을 채택
- API 호출 간 0.3초 딜레이 적용 (과호출 방지)

**수집할 필드**
- 식당명 (title 태그의 HTML 제거)
- 대분류 (category 필드에서 한식/중식/일식/양식/분식 등으로 매핑)
- 주소 (address)
- 네이버지도 URL (link)
- 기준 위치(서울 중구 세종대로 67, 위도 37.5663, 경도 126.9779)로부터
  직선 거리 계산 후 도보 소요시간으로 변환 (도보 속도 80m/분 기준, "약 N분" 형식)
- API 응답에서 위도/경도가 제공되는 경우 mapx, mapy 필드 활용
  (네이버 API의 mapx, mapy는 카텍 좌표계이므로 WGS84로 변환 후 거리 계산)

**결과 저장**
- 검색 결과를 찾지 못한 식당은 로그에 출력 후 건너뜀
- 중복 제거 후 restaurants.json 파일로 저장
- 도보 소요시간 기준 오름차순 정렬
- 수집 완료 후 총 몇 개 수집됐는지 출력

## STEP 2. Flask 웹 앱 구현 (로컬)

아래 구조로 Flask 웹 앱(app.py)을 작성해줘.

**탭 구성**

[전체 목록] 탭
- restaurants.json을 읽어서 전체 식당 목록을 표로 표시
- 표 컬럼: 식당명 / 대분류 / 도보 소요시간 / 네이버지도 링크(클릭 시 새 탭 오픈)

[추천] 탭
- 상단에 "오늘의 추천 뽑기" 버튼
- 버튼 클릭 시 전체 목록 중 무작위 3개를 동일한 표 레이아웃으로 표시

**디자인 요구사항**
- 깔끔하고 모던한 UI
- 탭 전환은 JavaScript로 구현
- 반응형 레이아웃 (모바일에서도 보기 좋게)
- 네이버 지도 링크는 버튼 형태로 표시
